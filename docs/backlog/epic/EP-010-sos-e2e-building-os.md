# EP-010: System-of-Systems E2E against Building OS

**Status:** Draft
**Priority:** P0

## Goal

EP-009 proves the connectors against real protocol stacks but terminates at a **mock** Building OS Ingress. This epic closes the loop against the **real** Building OS stack using the existing `building-os-e2e-test` SoS test framework (M1–M8): a single `pointlist.csv` drives model import (OxiGraph), simulator generation, telemetry ingest (TimescaleDB), semantic read, and control round-trip. Today that framework's "SoS gateway" is a **mock MQTT publisher** (`adapters/mqtt-publisher`). This epic makes **nexus-gateway the production SoS gateway** in that framework: the same point list flows through nexus's real pipeline — BACnet/OPC-UA connectors → JetStream → Normalizer (`local_id → point_id`) → Store-and-Forward → Building OS — and control round-trips back through Egress + the Command Channel.

```
mvp-pointlist.csv ──┬─▶ Building OS model import (OxiGraph)                [M2]
                    ├─▶ simulator config ─▶ bacnet-sim / opcua-sim        [M3]
                    └─▶ nexus Point List (synced)                         (ADR-0003)
 bacnet-sim/opcua-sim ─▶ nexus connectors ─▶ EVENTS ─▶ Normalizer ─▶ S&F ─▶ Building OS Ingress ─▶ TimescaleDB   [M4]
 Building OS API  GET /telemetries/hot?pointId=…                          [M5]
 Building OS Egress ─▶ Command Channel ─▶ connector write ─▶ simulator    [M7/M8: point_id ↔ local_id]
```

## Acceptance Criteria

- [ ] `mvp-pointlist.csv` (or its SBCO TTL) is the **single source of truth**: Building OS model import, simulator generation, and nexus's synced Point List all derive from it; nexus's Point List format is reconciled to consume it (CSV/TTL ingest or a generated bridge), not a hand-maintained parallel JSON.
- [ ] The native addresses in `pointlist.csv` (`local_id`, `object_type_bacnet`, `instance_no_bacnet`) match what the simulators expose and what the connectors poll, so `local_id → point_id` resolution in the Normalizer is exercised against real values (not fixtures).
- [ ] **Ingest E2E (M4-equivalent):** telemetry generated by the simulators flows through nexus's full pipeline into Building OS and is queryable in TimescaleDB; the ingestion contract nexus uses (gRPC `GatewayIngress/StreamTelemetry` vs the validated-telemetry MQTT path the mock publisher uses) is fixed in **ADR-0009 (Building OS ingestion contract)**.
- [ ] **API read E2E (M5):** every ingested point is retrievable via `GET /telemetries/hot?pointId=…` with the canonical `point_id` that nexus's Normalizer assigned, proving identity alignment between gateway and Building OS (`(gateway_id, point_id)` ownership, ADR-0001).
- [ ] **Control round-trip E2E (M8):** a Building OS Egress control request for a `writable=true` point round-trips `point_id → local_id` through nexus's Egress + Command Channel to the connector write handler, the simulator state changes, and the resulting reading flows back up; non-writable points are rejected at the gate (M7).
- [ ] The run integrates with the framework's reporting (`reports/<test_run_id>/`, HTML visualization) so nexus's pipeline nodes are colored by pass/fail alongside the rest of the SoS pipeline.
- [ ] A documented one-command (or scripted) bring-up launches the real Building OS OSS stack + simulators + nexus-gateway and runs the milestone suite; gaps that can't run in CI (BACnet broadcast, Keycloak) are marked manual/opt-in.
