# EP-010: System-of-Systems E2E against Building OS

**Status:** Prod
**Priority:** P0

## Goal

EP-009 proves the connectors against real protocol stacks but terminates at a **mock** Building OS Ingress. This epic closes the loop against the **real** Building OS stack using the existing `building-os-e2e-test` SoS test framework (M1–M8): a single `mvp-pointlist.csv` drives model import (OxiGraph), simulator generation, telemetry ingest (TimescaleDB), semantic read, and control round-trip. Today that framework's "SoS gateway" is a **mock MQTT publisher** (`adapters/mqtt-publisher`). This epic makes **nexus-gateway the production SoS gateway** in that framework: the same point list flows through nexus's real pipeline — BACnet/OPC-UA connectors → JetStream → Normalizer (`local_id → point_id`) → Store-and-Forward → Building OS — and control round-trips back through Egress + the Command Channel.

```
mvp-pointlist.csv ──┬─▶ Building OS model import (OxiGraph)                [M2]
                    ├─▶ simulator config ─▶ bacnet-sim / opcua-sim        [M3]
                    └─▶ nexus Point List (synced)                         (ADR-0003)
 bacnet-sim/opcua-sim ─▶ nexus connectors ─▶ EVENTS ─▶ Normalizer ─▶ S&F ─▶ Building OS Ingress ─▶ TimescaleDB   [M4]
 Building OS API  GET /telemetries/hot?pointId=…                          [M5]
 Building OS Egress ─▶ Command Channel ─▶ connector write ─▶ simulator    [M7/M8: point_id ↔ local_id]
```

## Acceptance Criteria

- [ ] `mvp-pointlist.csv` (or its SBCO TTL) is the **single source of truth**: Building OS model import, simulator generation, and nexus's synced Point List all derive from it; nexus's Point List format is reconciled to consume it (CSV/TTL ingest or a generated bridge), not a hand-maintained parallel JSON.
- [ ] The native addresses in `mvp-pointlist.csv` (`local_id`, `object_type_bacnet`, `instance_no_bacnet`) match what the simulators expose and what the connectors poll, so `local_id → point_id` resolution in the Normalizer is exercised against real values (not fixtures).
- [ ] **Ingest E2E (M4-equivalent):** telemetry generated by the simulators flows through nexus's full pipeline into Building OS and is queryable in TimescaleDB. nexus delivers via the **canonical gRPC `GatewayIngress/StreamTelemetry`** interface (Building OS's `gateway_ingress.proto`); `GatewayIngressService` resolves enrichment and republishes to `building-os.validated.telemetry` internally, so the same `telemetry-consumer → TimescaleDB` path the mock MQTT publisher exercised is reached through the real front door. The mock `adapters/mqtt-publisher` is bypassed, not reimplemented.
- [ ] **API read E2E (M5):** every ingested point is retrievable via `GET /telemetries/hot?pointId=…` with the canonical `point_id` that nexus's Normalizer assigned, proving identity alignment between gateway and Building OS (`(gateway_id, point_id)` ownership, ADR-0001).
- [ ] **Control round-trip E2E (M8):** a Building OS Egress control request for a `writable=true` point round-trips `point_id → local_id` through nexus's Egress + Command Channel to the connector write handler, the simulator state changes, and the resulting reading flows back up; non-writable points are rejected at the gate (M7).
- [ ] The run integrates with the framework's reporting (`reports/<test_run_id>/`, HTML visualization) so nexus's pipeline nodes are colored by pass/fail alongside the rest of the SoS pipeline.
- [ ] A documented one-command (or scripted) bring-up launches the real Building OS OSS stack + simulators + nexus-gateway and runs the milestone suite; this heavy SoS run is **manual/nightly**, not nexus's per-PR CI. The M7/M8 control gate's Keycloak is the **Building OS stack's** (`building-os.keycloak`, realm `buildingos`) brought up with that stack — nexus's CI does not provision it; nexus's own simplified Keycloak (Admin API, EP-004) is separate and unrelated. BACnet broadcast E2E that needs host networking is also manual if the runner can't host it.

## Dependencies

- EP-009 Simulator Integration — provides the simulator wiring and protocol-level E2E this epic builds on.
- EP-003 Normalizer/Uplink, EP-005 Control Path, EP-006 Point List Sync — the pipeline stages under SoS test.
- EP-008 FEAT-032 (transport security) — the real Building OS link should use the secured gRPC path, not insecure.
- **External:** `gutp-building-os-oss` (OSS stack: OxiGraph, TimescaleDB, API). nexus replaces `adapters/mqtt-publisher` as the SoS gateway. **The OSS dev compose does not start the gRPC front doors** — the test topology must add ConnectorWorker's `GatewayIngressService` (telemetry ingress port, internal-only by default) and GatewayBridge's `GatewayEgressService` (control egress), and run telemetry-consumer with the timescale warm-store profile. No Envoy in dev ⇒ nexus dials h2c directly (ADR-0007 `--insecure` dev path).
- **Resolved (grill 2026-06-13/14):**
  - Telemetry ingestion uses the canonical gRPC `GatewayIngress/StreamTelemetry`; `building-os.validated.telemetry` is the internal downstream bus subject, not an alternative gateway contract (no ADR-0009).
  - Point List source follows ADR-0003 precedence (provisioning sync > CSV/file bootstrap); FEAT-041 implements both tiers from one CSV — now **AFK**, no open contract.
  - **Scope:** EP-010 SoS is **BACnet-first** because `mvp-pointlist.csv` carries only BACnet native columns. OPC-UA stays at EP-009 (protocol-level, mock Building OS) for the MVP; OPC-UA SoS entry waits on the e2e CSV gaining OPC-UA points + the `ns=2;s=<point_id>` NodeId projection (FEAT-039 follow-on, not in this epic's MVP slice).
- **No remaining HITL gates** in this epic after the above.
